# 软件体系结构速成课程包生成计划

## Summary

目标是在 `docs/软件体系结构/` 下生成一套 Markdown 课程包，帮助有软件工程背景但架构基础不扎实的开发者，在 AI 辅助开发场景下理解和设计中大型项目架构。

课程不写成教材，不堆术语；每个概念都要回答“为什么需要它”，并持续结合 RAG、论文解析、Agent、长期记忆、风格学习系统举例。最终计划文件位置为 `docs/软件体系结构/PLAN.md`。

## File Roles And Chapter Plan

| 文件 | 学习优先级 | 作用 |
|---|---|---|
| `README.md` | 必读 | 课程包入口，说明适合谁、怎么学、如何和 Codex 协作。 |
| `00-学习路线.md` | 必读 | 给出快速入门、项目实战、系统补课三条路线。 |
| `01-架构到底在解决什么问题.md` | 必学 | 建立架构意识：架构是管理复杂度、变化、协作和风险。 |
| `02-高内聚与低耦合.md` | 必学 | 学会判断模块应该怎么拆，避免 AI 协作时越改越乱。 |
| `03-MVC与前后端职责边界.md` | 必学 | 学会把 UI、交互状态、业务逻辑拆开。 |
| `04-分层架构.md` | 必学 | 学会入口层、应用层、领域层、基础设施层的职责边界。 |
| `05-事件驱动架构.md` | 建议学 | 学会处理异步任务、状态流转和长流程解耦。 |
| `06-六边形架构.md` | 必学 | 学会让核心业务不依赖框架、数据库和 LLM SDK。 |
| `07-插件化架构.md` | 建议学 | 学会为解析器、检索器、工具、记忆策略设计扩展点。 |
| `08-RAG系统架构.md` | 必学 | 学会从文档进入系统到生成回答的完整链路。 |
| `09-Agent系统架构.md` | 必学 | 学会 Agent 状态、工具边界、计划执行循环和失败恢复。 |
| `10-长期记忆与风格学习系统架构.md` | 必学 | 学会记忆写入、检索、遗忘、冲突和风格样本沉淀。 |
| `11-AI辅助开发中的架构协作.md` | 必学 | 学会把架构设计变成 Codex 可执行任务。 |
| `12-架构设计实战模板.md` | 必学 | 提供可直接复用的架构设计文档模板。 |
| `CHECKLIST.md` | 必读 | 用于学习复盘和真实项目架构评审。 |

## Generation Order

1. 先生成 `PLAN.md`，固定课程范围和质量标准。
2. 生成 `README.md` 和 `00-学习路线.md`，建立学习入口。
3. 生成基础架构章节：`01` 到 `04`。
4. 生成进阶架构章节：`05` 到 `07`。
5. 生成 AI 项目专题章节：`08` 到 `10`。
6. 生成 AI 协作与实战模板：`11`、`12`。
7. 最后生成 `CHECKLIST.md`，统一验收和复习标准。

## Quality Standards

- 每章必须说明：这个概念解决什么痛点、什么时候需要、什么时候不需要。
- 每章必须至少包含 2 个来自 RAG / Agent / 长期记忆 / 风格学习系统的例子。
- 每章必须包含“坏设计长什么样”和“更好的拆法”。
- 每章必须包含一个可执行产物：清单、模板、架构图、模块拆分表或 Codex 提示词。
- 不使用考试式定义开头；优先从真实开发困境切入。
- 不堆术语；出现术语时必须用项目场景解释它为什么存在。
- Markdown 应适合长期保存：标题清晰、层级稳定、示例可复用。
- 所有内容默认面向真实项目开发和 Codex 协作，不面向背诵考试。
- 不修改代码接口、运行逻辑或项目配置；本课程包只新增 Markdown 文档。
